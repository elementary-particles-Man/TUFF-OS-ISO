以下は、**TUFF-OS** における **バックアップからのリストア（復元）手順** の詳細説明です。

この手順は、以下のいずれかの状況で必要になります：

- Recovery PIN紛失による再インストール後
- 物理ディスク全損・交換後
- 意図的なデータリセット後
- テスト環境から本番環境への移行時

**重要警告（必ず最初に読んでください）**

- リストア対象の**既存データはすべて上書き**されます。  
  → リストア前に**現在のデータを別の場所へ完全バックアップ**してください。
- バックアップ元は**TUFF-OS外の安全なストレージ**（外部HDD、NAS、クラウド暗号化領域など）であること。
- リストア中は**電源を絶対に切らない**でください（電断時は自動ロールバック機能が働く設計ですが、念のためUPS推奨）。

### バックアップリストア手順（ステップ・バイ・ステップ）

#### Step 1: バックアップデータの準備確認

1. バックアップ元を確認  
   - 推奨形式：**TUFF-FS互換のRAWイメージ**（`dd` で取得したもの）または **tuffutl export** で出力した論理バックアップ  
   - ファイル例：  
     - `tuff_backup_20260322.tar.zst`（論理バックアップ）  
     - `jbod_backup.img`（物理JBOD全体イメージ）

2. バックアップの整合性チェック  
   ```bash
   # 論理バックアップの場合
   tuffutl backup verify --file tuff_backup_20260322.tar.zst

   # 物理イメージの場合
   sha256sum jbod_backup.img
   # 事前に記録したハッシュ値と一致するか確認
   ```

3. バックアップを**リストア用ストレージ**（外部HDDなど）にコピーしておく

#### Step 2: TUFF-OS再インストール（PIN紛失時・全損時）

1. USBインストーラからブート  
   - BIOS/UEFIでUSBを最優先に設定

2. 「**完全再インストール（データ全消去）**」を選択  
   - 警告ダイアログを2回確認

3. 対象ディスク選択  
   - SSD（システム領域）  
   - HDD群（JBODプール：最低3台、推奨5台）

4. 新しいRecovery PINが生成・表示される  
   → **紙に記録** + **USBメモリに暗号保存**（忘れずに！）

5. インストール完了 → 再起動 → 初回ブートでGenesis再初期化

#### Step 3: リストア実行

**方法A: 論理バックアップからのリストア（推奨・高速）**

```bash
# 1. バックアップファイルをTUFF-OS内にコピー
# （USBメモリや外部HDDからマウントしてコピー）

# 2. リストアコマンド実行
tuffutl backup restore \
  --file /mnt/external/tuff_backup_20260322.tar.zst \
  --target /data \
  --commit auto \
  --verify

# 主なオプション解説
--file        バックアップファイルパス（必須）
--target      復元先ルートパス（例: /data）（必須）
--commit auto  復元後自動コミット（推奨）
--verify       復元後に整合性チェック（推奨）
--dry-run      事前シミュレーション（安全確認用）
```

**方法B: 物理イメージからのリストア（全ディスク完全復元）**

```bash
# 1. イメージファイルを全HDDにddで書き込み
# 注意：対象HDDの全データが消去されます

dd if=jbod_backup.img of=/dev/sdb bs=4M status=progress conv=fsync
dd if=jbod_backup.img of=/dev/sdc bs=4M status=progress conv=fsync
# ... 全HDDに対して繰り返し

# 2. TUFF-OS再起動後、整合性チェック
tuffutl sys fsck --repair

# 3. 状態確認
tuffutl sys status
```

#### Step 4: リストア後の確認・最終調整

1. **システム状態確認**
   ```bash
   tuffutl sys status --detail
   ```
   → Genesis Valid、3N Healthy、Isolation Inactive を確認

2. **ユーザー再作成・ログイン**
   ```bash
   tuffutl user add admin --password "NewStrongPass123!"
   tuffutl sys login admin --password "NewStrongPass123!"
   ```

3. **TagGroupMask再適用**
   機密フォルダにタグを再付与
   ```bash
   tuffutl fs tag add /data/secret "社外秘" "財務"
   ```

4. **ネットワーク設定再適用**
   ```bash
   tuffutl nw blacklist refresh
   tuffutl nw aiserverlist add https://trusted.ai --password "xxx"
   ```

5. **コミット実行**
   ```bash
   tuffutl fs commit --target /
   ```

6. **最終確認**
   - ファイル一覧・内容確認
   - `tuffutl log tail` で異常ログがないかチェック

### 注意点・リスクまとめ

| 注意点                       | リスク内容                           | 予防策 |
|------------------------------|--------------------------------------|--------|
| リストア先ディスク選択ミス   | 間違ったドライブを上書き             | デバイス名を3回確認 |
| バックアップファイル破損     | リストア失敗 → データ完全喪失       | リストア前に `tuffutl backup verify` |
| リストア中電源断             | 部分リストアで不整合                 | UPS必須 + 自動ロールバック機能に依存 |
| 新Recovery PINの紛失         | 次回障害時に再インストール必要       | PINを複数箇所に安全分散保存 |

**最重要ルール**  
「**リストアは最後の手段**」です。  
PIN紛失を防ぐことが最優先。  
PINは**命より大切に管理**してください。

ご不明な点や、特定のバックアップ形式（例：論理 vs 物理）での追加手順が必要でしたら、遠慮なくお知らせください。🛡️
